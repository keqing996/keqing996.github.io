    目标是搭建一个不带前端主流框架的electron环境，也就是没有vue和react等。包含的主流组件为：

    • Electron自不必说
    • Electron-forge来打包（packager或者builder也都可以，一样的）
    • Typescript，用TypeScript有两个原因，首先当然是TypeScript好使，其次是，Electron至今都不能很好的支持ESM，因此直接写js的话，在包的引入上需要用require，但是如果require的包又是es语法用了export或者import，就会报错（比如echarts的引入），非常难受。因此用Typescript写，把target设置成commonjs来转一手。
    • webpack，打包用。

    在空文件夹下，首先npm init。生成package.json。

    npm init

    一 安装各种node包

    首先是electron。会自动创建node_modules和package-lock.json。

    npm install --save-dev electron

    接下里，安装webpack。

    npm install --save-dev webpack webpack-cli webpack-merge webpack-node-externals

    注意需要安装一下webpack-merge，因为electron需要单独打包主进程和泫然进程，一些共用的配置会拿出来，需要webpack-merge帮忙合并。webpack-node-externals用来在打包过程中排除掉node_modules。

    接下来，安装typescript。

    npm install --save-dev typescript

    以及需要用到的typescript的扩展。这些是为了引入对应的类型，让typescript正确识别不报错。

    npm install --save-dev @types/node @types/webpack

    过去还需要@types/electron，但是现在electron自带了，就不需要了。 接下来是webpack打包过程的一些loader。

    npm install --save-dev css-loader file-loader node-loader style-loader ts-loader

    其中：

    • css-loader+style-loader：解决css打包
    • node-loader：解决native的node文件打包
    • ts-loader：解决typescript文件打包
    • file-loader：解决其他乱七八糟文件打包 接下里是webpack打包过程中用来打包html的plugin。
    npm install --save-dev html-webpack-plugin

    不打包的话，上面这些就够了。

    二 让流程跑起来

    流程上是这样，在src目录下，用ts+html+css把electron正常的主进程和渲染进程逻辑写好。然后用一揽子的webpack配置把他们生成到dist目录，最后把package.json中的入口调整为dist下编译出来的js文件。除了dist文件以外，其他的文件包括src，在打包的时候都不需要。

    1 基础逻辑文件

    首先创建如下的文件目录。

    在main.ts里写主进程的逻辑，就是官方文档中标准写法的ts版本。

    import { BrowserWindow, app, ipcMain } from 'electron';
import path from "path";

let mainWindow: BrowserWindow | null = null;

function createMainWindow() {
    if (mainWindow)
        return;

    mainWindow = new BrowserWindow({
        webPreferences: {
            // 打开这选项，可以在渲染进程使用node
            // 否则就只能在主进程使用node，所有消息靠进程间通信实现
            // 但是electron官方非常不推荐，这样不太安全
            nodeIntegration: true,
        },
        height: 800,
        width: 1600
    });

    // 这里的路径其实是webpack生成之后的指向路径，保持生成前后名字不变的话，就还好
    mainWindow.loadFile(path.resolve(__dirname, `../render/index.html`));
    mainWindow.on('close', () => mainWindow = null);

    mainWindow.webContents.openDevTools()
}

app.on('ready', () => {
    createMainWindow();
});

app.on('window-all-closed', () => {
    if (process.platform !== 'darwin') {
        app.quit();
    }
});

app.on('activate', () => {
    createMainWindow();
});

    接下来是index.html，随便写点。

    <!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <title>ElectronApp</title>
</head>
<body>
    <h1>Hello Electron</h1>
    <p>一个幽灵，共产主义的幽灵，在欧洲游荡。为了对这个幽灵进行神圣的围剿，旧欧洲的一切势力，教皇和沙皇、梅特涅和基佐、法国的激进派和德国的警察，都联合起来了。</p>
    <div id="app"></div>
    <div id="app"></div>
</body>
</html>

    注意这里不要插入render.js，webpack打包会帮忙插入。接下来是css，随便写点。

    h1 {
    color: blueviolet;
    text-align:center;
}

p {
    color: brown;
    font-family:"Times New Roman";
    font-size:20px;
}

    最后是render.ts。

    import './style.css' // 引入一下让webpack识别到来打包

    这里是后续写业务逻辑的地方，只是基础环境的话，这里什么都不做，但是一定要写上上面这句话。没有这句话，webpack打包无法正确引入这个css。

    2 配置tsconfig

    直接根目录创建一个tsconfig.json，或者用下面的命令行创建一个。

    tsc init

    配置一下typescript的编译，具体含义就不一一解释了，可以参考typescript的文档。

    {
    "compilerOptions": {
      "target": "ES2018",
      "module": "CommonJS",
      "lib": [ "dom", "esnext" ],
      "strict": true,
      "noImplicitReturns": true,
      "moduleResolution": "Node",
      "esModuleInterop": true,
      "allowSyntheticDefaultImports": true,
      "allowJs": true,
      "resolveJsonModule": true
    },
    "exclude": [ "node_modules" ],
    "include": [ "src/main", "src/renderer" ]
  }

    3 配置webpack

    webpack需要单独配置主进程和渲染进程，一些共同的配置单独抽一个js，然后用merge来结合。 首先根目录创建webpack.base.config.js，写一些基本的规则。

    module.exports = {  
    module: {  
        rules: [  
            // typescript  
            {  
                test: /\.ts$/,  
                use: 'ts-loader',  
                exclude: /node_modules/  
            },  
            // css (style loader在前 css loader在后)  
            {  
                test:/\.css$/,  
                use: [ 'style-loader', 'css-loader'],  
                exclude: /node_modules/  
            },  
            // .node native (这里例子中用不到)
            {  
                test: /\.node$/,  
                exclude: /node_modules/,  
                use: 'node-loader'   
			}  
        ]  
    }  
};

    接下里写主进程的webpack配置，根目录新建文件webpack.main.config.js，内容如下。把输出目录设置为dist（目前没有，一会会自动生成）。

    const path = require('path');  
const webpack = require('webpack');  
const { merge } = require('webpack-merge');  
const nodeExternals = require('webpack-node-externals');  
const webpackBaseConfig = require('./webpack.base.config');  
  
module.exports = merge(webpackBaseConfig, {  
    mode: 'development',  
    target: 'node',  
    entry: path.join(__dirname, 'src/main/main.ts'),  
    output: {  
	    // 注意输出目录
        path: path.join(__dirname, 'dist/main'),  
        filename: 'main.js'  
    },  
    externals: [nodeExternals()],  
    plugins: [  
        new webpack.EnvironmentPlugin({  
            NODE_ENV: 'development'  
        })  
    ],  
    node: {  
        __dirname: false,  
        __filename: false  
    }  
});

    接下来是渲染进程，文件webpack.render.config.js。内容和main的差不多，多一个html的插件。

    const path = require('path');
const webpack = require('webpack');
const { merge } = require('webpack-merge');
const HtmlWebpackPlugin = require('html-webpack-plugin')
const webpackBaseConfig = require('./webpack.base.config');

module.exports = merge(webpackBaseConfig, {
    mode: 'development',
    target: 'electron-renderer',
    entry: path.join(__dirname, 'src/render/render.ts'),
    output: {
        path: path.join(__dirname, 'dist/render'),
        filename: 'render.js'
    },
    plugins: [
        new webpack.EnvironmentPlugin({
            NODE_ENV: 'development'
        }),
        new HtmlWebpackPlugin({
            inject: 'body',
            scriptLoading: 'defer',
            minify: false,
            filename: 'index.html',
            template: path.join(__dirname, 'src/render/index.html')
        }),
    ],
});

    最后一步，配置package.json的入口，让webpack可以run起来。在package.json里增加：

    "webpack-main": "webpack --config webpack.main.config.js",
"webpack-render": "webpack --config webpack.render.config.js"

    两个命令分别用webpack生成主进程和渲染进程。接下来试一试，在命令行里分别输入：

    npm run webpack-main
npm run webpack-render

    都成功的话，可以看到生成的dist目录。

    4 调整入口，运行

    在package.json中把main调整为dist的生成js。并且在脚本中增加一个electron运行。

    在命令行输入下面的命令来运行electron。

    npm start

    一切顺利的话，就可以看到界面了（下面的截图左侧是当前的文件结构，方便对照）。console会有一个warning，因为在主进程起BrowserWindow的时候，设置了nodeIntegration，官方认为这样不安全，因为会报warning。如果不是对外发布app的话，无视就好。如果是对用户的正式app，推荐还是按官方的做法做。主进程跑逻辑，ipc通知渲染进程更新。

    三 打包

    electron-forge是官方提供的一个打包工具（其实就是electron-packager套壳）。首先安装一下。

    npm install --save-dev @electron-forge/cli

    接下来执行下面这句话，让forge自动配置一下项目。

    npm exec --package=@electron-forge/cli -c "electron-forge import"

    它会做三个事情。 一个是帮忙装上其他的forge依赖。

    二是改动start入口，从forge进入，以及帮忙生成另外两个打包的入口。

    三是生成一个打包配置文件forge.config.js。打开这个文件，做一些基础改动。

    module.exports = {  
  packagerConfig: {  
    appVersion: "1.0.0",  
    platform: "win32",  
    name: "BaseElectronEnv",   
    asar: true,  
    ignore:[  
        "src",  
        "node_modules",  
        "forge.config.js",  
        "tsconfig.json",
        "webpack.base.config.js",  
        "webpack.main.config.js",  
        "webpack.render.config.js",  
    ],  
    win32metadata :{  
      ProductName: "BaseElectronEnv",  
      FileDescription: "Base Electron Env"  
    }  
  },  
  rebuildConfig: {},  
  makers: [  
	// 随便写两个例子
    {  
      name: '@electron-forge/maker-squirrel',  
      config: {},  
    },  
    {  
      name: '@electron-forge/maker-zip',  
      config: {},  
    }  
  ],  
};

    参数就不详细解释了。可以先npm start一下确认可以正常启动。接下里依次输入（记得package.json中的Auther和Description要写一下，否则打不出来）。

    npm run package
npm run make

    就完成打包了。输出在根目录的out文件夹。
